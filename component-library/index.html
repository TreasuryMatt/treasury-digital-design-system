Buttons

All buttons use the .usa-button base class plus a variant modifier. Base styles: 15px, 700 weight, 2px border, 4px radius, 10px / 20px padding.

Variants

<!-- Primary action -->
<button class="usa-button usa-button--primary">Save</button>

<!-- Secondary / neutral -->
<button class="usa-button usa-button--secondary">Cancel</button>

<!-- Outlined -->
<button class="usa-button usa-button--outline">Export</button>

<!-- Confirm / approve -->
<button class="usa-button usa-button--success">Approve</button>

<!-- Destructive action -->
<button class="usa-button usa-button--danger">Delete</button>

<!-- Link-style, no border or background -->
<button class="usa-button usa-button--unstyled">View details</button>

Small Buttons — .usa-button--sm

Reduced to 13px / 6px–14px padding. Use inside tables, card headers, toolbars, and filter bars.

<button class="usa-button usa-button--primary usa-button--sm">View</button>
<button class="usa-button usa-button--outline usa-button--sm">Export</button>

Disabled State

<!-- The disabled attribute reduces opacity to .5 and blocks pointer events -->
<button class="usa-button usa-button--primary" disabled>Disabled</button>

Form Controls

Inputs use a 2px border in var(--usa-base) with a vivid blue focus ring. Always pair an input with a .usa-label. Hint text and error messages sit below the input.

USWDS Icons

All icons come from the USWDS SVG sprite. Download it once and reference icon names via <use href="/assets/img/sprite.svg#icon-name">. Never use emoji as icons anywhere in the UI.

<!-- One-time setup: download the USWDS sprite -->
curl -L "https://cdn.jsdelivr.net/npm/@uswds/uswds/packages/uswds-core/src/img/sprite.svg" \
     -o public/assets/img/sprite.svg

<!-- Usage: inline SVG with sprite reference -->
<svg class="usa-icon" aria-hidden="true" focusable="false">
  <use href="/assets/img/sprite.svg#search"></use>
